—— The Problem

Manual Cluster Provisioning Doesn't Scale

  • Slow time-to-cluster. Weeks of tickets, approvals, and manual configuration before a team gets a working environment.
  • Configuration drift. No two clusters are alike. Security patches applied inconsistently. Debugging becomes archaeology.
  • No audit trail. Who changed what, and when? Manual changes vanish into the ether — invisible to compliance audits.
  • Toil compounds. Every new cluster multiplies operational burden. The team that manages 5 clusters cannot manage 50 the same way.
The challenge: infrastructure at enterprise scale demands a fundamentally different approach.
—— The Insight

What If Every Cluster Was a Pull Request?

A new cluster should be as simple as committing YAML to a Git repository. Argo CD syncs the desired state. RHACM provisions the infrastructure. ApplicationSets bootstrap Day 2 configuration automatically. No tickets. No manual steps. Just code review and merge.

—— Phase 1 · Hub Setup

Install the Operators, Wire the Integration

1 ACM Operator

Install advanced-cluster-management from the release-2.16 channel. Deploy the MultiClusterHub CR to activate Hive, Placement, and the full management stack.

2 GitOps Operator

Install openshift-gitops-operator from the latest channel (v1.20). Configure the ArgoCD CR with ApplicationSet support, route access, and ACM resource health checks.

3 The Critical Glue

Create the ManagedClusterSet, bind it to the GitOps namespace, and configure Placement. This lets Argo CD discover and target spoke clusters securely.

—— Phase 3 · Day 1 Provisioning

A Cluster Is Four Custom Resources

ClusterDeployment

Cluster name, base domain, AWS region, ClusterImageSet (OCP 4.20), credential secret references.

MachinePool

Worker node instance type, replica count, root volume configuration. Dev: 2x m5.xlarge. Prod: 5x m5.2xlarge with autoscaling.

ManagedCluster

Registers the cluster in ACM. Carries the labels that drive everything: environment, region, pci-compliant.

KlusterletAddonConfig

Deploys management agents: application manager, policy controller, search collector, cert policy controller.

Commit these manifests → Argo CD syncs to Hub → Hive provisions on AWS → cluster reports Ready.

Prod Cluster

Labels are the control plane. The pci-compliant: true label on prod automatically triggers stricter NetworkPolicy enforcement through ApplicationSets. No manual configuration required.

—— Best Practices

Enterprise-Grade from Day One

  • Secrets management. SealedSecrets or External Secrets Operator with Vault / AWS Secrets Manager. Never commit credentials in plain text.
  • selfHeal: true. Argo CD automatically reverts any manual drift on managed clusters. The Git repo is always the authority.
  • Shift-left validation. CI pipelines run kustomize build and kube-linter on every PR. Catch misconfigurations before they reach any cluster.
  • Label-driven targeting. Design your ManagedCluster label taxonomy upfront. Labels control which configs reach which clusters — they are your fleet's control plane.
